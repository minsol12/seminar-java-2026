### 요구사항

1. **적금 계좌**와 **입출금 계좌** 두 종류의 계좌를 구현한다
    - 적금 계좌는 입금, 잔액 조회만 가능하다
    - 입출금 계좌는 입금, 출금, 이체, 잔액 조회가 가능하다
2. 아래 상황에서 적절한 예외를 던진다
    - 0 이하의 금액을 입력했을 때
    - 잔액보다 큰 금액을 출금/이체 시도했을 때
3. 예외 발생 시 프로그램이 중단되지 않아야 한다.


-----
### Step 1
클래스 - 계좌, 적금 계좌, 입출금 계좌
필드 - 잔액
메서드 - 입금, 출금, 이체, 잔액 조회

### Step 2
적금 계좌와 입출금 계좌
공통점 - 입금, 잔액 조회 가능 - 부모 또는 인터페이스
차이점 - 입출금 계좌는 출금, 이체도 가능 - 자식 클래스에서만

### Step 3 (상태(필드), 매개변수, 행동(메서드) 분리)
계좌 - 잔액 - 적금 계좌, 입출금 계좌
적금 계좌 - 잔액 - 입금할 금액 - 잔액 조회, 입금
입출금 계좌 - 잔액 - 입금할 금액, 출금할 금액, 이체할 금액, 이체 받을 사람 이름 - 잔액 조회, 입금, 출금, 이체

### Step 4 (메서드)
deposit - 입금
- 매개변수: 입금할 금액
- 반환값: 없음. (메서드 안에서 출력 처리)

withdraw - 출금
- 매개변수: 출금할 금액
- 반환값: 없음. (메서드 안에서 출력 처리)

transfer - 이체
- 매개변수: 이체할 금액, 이체 받을 사람 이름
- 반환값: 없음. (메서드 안에서 출력 처리)

inquiry - 잔액 조회
- 매개변수: X
- 반환값: 없음. (메서드 안에서 출력 처리)

### Step5 (예외)
- 출금 시도 시 잔액 부족 - IllegalArgumentException
  - 던지는 곳 - 출금 메서드 안
  - 잡는 곳 - main
- 이체 시도 시 잔액 부족 - IllegalArgumentException
  - 던지는 곳 - 이체 메서드 안
  - 잡는 곳 - main
- 0 이하 금액을 입력했을 때 - IllegalArgumentException
  - 던지는 곳 - 입금, 출금, 이체 메서드 안
  - 잡는 곳 - main

### Step6
공통 부모 (계좌 클래스) - 추상 클래스 (이유: 필드(balance)를 가져야 하기 때문!)
자식 (적금 계좌 클래스, 입출금 계좌 클래스)

### 결정사항
- 이체는 송금자 잔액만 차감한다 (수신자 잔액은 관리 X)
- 계좌 객체는 소유자 이름을 모른다 (Main에서 변수 이름으로 식별)
